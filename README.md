Organ Donation Management System

A comprehensive **Database Management System (DBMS)** project developed using **MySQL, SQL, and PL/SQL** to model the complete organ donation and transplantation workflow. The system provides a structured and normalized database for managing donors, patients, doctors, hospitals, organ availability, and transplant transactions while ensuring data integrity through constraints, triggers, stored procedures, functions, and transaction management.

The project demonstrates the application of core database concepts, including Entity-Relationship (ER) modeling, relational schema design, normalization up to Third Normal Form (3NF), complex SQL queries, views, PL/SQL programming, cursors, triggers, and ACID-compliant transactions. It aims to simulate a real-world organ donation management system by improving data organization, reducing redundancy, and automating critical business rules involved in organ allocation and transplantation.

This project was developed collaboratively as part of a two-member team for the Database Management Systems (DBMS) course at Thapar Institute of Engineering and Technology.

Both team members actively contributed throughout the project lifecycle, including requirement analysis, database design, normalization, SQL and PL/SQL implementation, testing, debugging, and documentation. The project reflects a shared effort in building a comprehensive and well-structured database solution.
